DevTools: JavaScript

Consola, errores comunes y breakpoints para entender el flujo.

// Fragmento listo para practicar en la consola
const demoButton = document.querySelector("#demoToggle");
const demoState = document.querySelector("#demoState");

const setDemoMode = (mode) => {
  console.log("Modo anterior:", demoState.dataset.mode);
  demoState.dataset.mode = mode;
  demoState.textContent = `Modo actual: ${mode}`;
};

demoButton.addEventListener("click", () => {
  const next = demoState.dataset.mode === "light" ? "dark" : "light";
  setDemoMode(next);
});
Tips rápidos:
  • Abre la consola y escribe `console.clear()` antes de cada prueba.
  • Rompe el script (por ejemplo, eliminando un paréntesis) y lee el error con calma.
  • Coloca un breakpoint en `setDemoMode` para inspeccionar `mode` paso a paso.
  • Vuelve a tu portfolio real y añade los mismos `console.log` al toggle del modo oscuro.

Git básico aplicado al portfolio

Inicializa, revisa estado y crea commits descriptivos.

Flujo general

git init
git status
git add index.html css/styles.css js/app.js
git commit -m "chore: versión inicial de UT5"

Después de ajustar estilos

git status
git add css/styles.css
git commit -m "style: mejora botón de WhatsApp"

Después de depurar JS

git status
git add js/app.js
git commit -m "fix: asegura toggle de modo oscuro"
