# 📘 Práctica Integrada 01_05 · Fundamentos Aplicados

## Qué bloque trabaja de verdad

Esta práctica cierra el primer tramo útil del curso:

- tipos de datos
- condicionales
- bucles
- funciones
- listas y diccionarios

No introduce teoría nueva. Sirve para comprobar si la base ya se puede usar con sentido dentro de un problema continuo.

## Qué debería demostrar el alumno al terminar

Un alumno que aprovecha bien este bloque debería poder:

- leer y modificar una lista de diccionarios con soltura
- localizar un elemento por id o por criterio simple
- validar antes de cambiar stock o estado
- separar búsqueda, modificación y presentación
- explicar el flujo del menú sin perderse

## Por qué este inventario está bien elegido

El escenario funciona porque obliga a hacer justo lo que interesa en este punto:

- guardar varios campos por elemento
- recorrer estructuras de datos
- buscar con criterio
- actualizar estado
- evitar errores lógicos básicos

## Qué conexión tiene con el temario

- Tema 01: tipos de datos y variables usadas en cada campo
- Tema 02: validaciones y decisiones antes de modificar
- Tema 03: bucles para menús y recorridos
- Tema 04: reparto en funciones pequeñas
- Tema 05: uso de listas y diccionarios como estructura principal

## Estrategia recomendada de resolución

Orden útil:

1. entender la estructura del inventario
2. resolver la búsqueda de producto
3. implementar altas y ventas
4. revisar validaciones
5. añadir estadísticas o filtros sencillos

## Qué conviene observar durante la corrección

- si reutiliza una función de búsqueda
- si valida antes de descontar stock
- si evita ids incoherentes
- si distingue entre mostrar datos y modificarlos
- si el flujo del menú se entiende al leerlo

## Errores frecuentes

- duplicar la lógica de búsqueda
- vender sin comprobar stock suficiente
- mezclar `input`, cálculo y presentación
- modificar el inventario base sin querer
- aceptar cantidades o precios absurdos

## Señales de que el bloque está bien dominado

La práctica ha cumplido su función cuando el alumno:

- deja de programar a golpes
- entiende por qué cada función existe
- detecta duplicaciones sin que se lo digan
- propone una mejora razonable sin salir del nivel

## Buen uso del laboratorio IA

Buenas ampliaciones:

- filtro por categoría
- actualización de precio
- ranking por valor total en stock

Malas ampliaciones:

- meter base de datos
- convertirlo en POO antes de tiempo
- hacer una aplicación enorme
