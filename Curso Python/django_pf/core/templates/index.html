Bienvenido a Django

Practica rutas, plantillas, JSON y manejo de errores en una sola app.

  • Rutas y parámetros
  • Formularios (POST)
  • API JSON
  • Manejo de errores y middleware
Consejo

Abre DevTools (F12) y mira la pestaña Network para ver cada petición.

1) Formulario (POST)

Método POST: envía datos al servidor sin mostrarlos en la URL.

{% csrf_token %}
<form action="/procesar/" method="post">
<input name="nombre" ...>
</form>
2) Rutas con parámetros (/ruta/valor)

Los parámetros van en la URL. Haz clic en los enlaces:

  • /saluda/Joaquin/ Parámetro: nombre (string)
  • /suma/3/5/ Parámetros tipados: int (3) + int (5)
path("saluda/<str:nombre>/", views.saluda)
def saluda(request, nombre): ...
3) Query Strings (?param=valor)

Los parámetros van DESPUÉS del ? sin cambiar la ruta.

  • /buscar/?q=django&page=1 Búsqueda: django, página: 1
  • /buscar/?q=python&page=2 Búsqueda: python, página: 2
q = request.GET.get("q", "")
page = request.GET.get("page", "1")
4) API JSON (GET/POST)

Endpoints que devuelven JSON (formato de datos ligero).

  • GET /api/echo/?q=hola Devuelve: {"ok": true, "echo": "hola"}
  • GET /api/health/ Estado de la API: {"status": "ok"}
  • GET /api/calculadora/suma/3/5/ Calculadora básica vía JSON
return JsonResponse({"ok": True, "echo": q})
5) Manejo de errores (HTTP 404, 500)

Prueba rutas que no existen o que generan errores:

  • 404 - No encontrado La ruta no existe en la app
  • 500 - Error intencional Provoca un error controlado
