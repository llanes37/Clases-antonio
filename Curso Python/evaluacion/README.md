# Evaluación del Curso

## Qué papel cumple esta carpeta

`evaluacion/` existe para responder una pregunta muy concreta:

**¿el alumno ya sostiene el bloque sin demasiado apoyo o todavía depende del temario y de la práctica guiada?**

Por eso esta carpeta no debería usarse al principio de un bloque, sino como cierre cuando el alumno ya ha visto teoría y ha pasado por práctica integrada.

## Diferencia entre temario, prácticas y evaluación

### Temario

Sirve para:

- introducir conceptos
- explicar sintaxis
- enseñar patrones
- practicar ideas aisladas

### `practicas/`

Sirve para:

- mezclar varios temas
- cometer errores con margen
- consolidar flujo y criterio
- acercarse a tareas más realistas

### `evaluacion/`

Sirve para:

- medir autonomía
- comprobar comprensión real
- detectar huecos del bloque
- cerrar una etapa con criterio claro

## Cuándo conviene usar estas evaluaciones

Secuencia recomendada:

1. terminar el bloque del temario
2. resolver la práctica integrada correspondiente
3. pasar a la evaluación
4. corregir con guía y feedback breve

## Qué caracteriza una buena evaluación en este curso

Una evaluación útil debería:

- responder a un bloque reconocible del temario
- usar un escenario comprensible
- medir varias destrezas a la vez
- obligar a ordenar el problema
- poder corregirse con criterio y sin improvisar
- tener versión profesor, versión alumno y guía

## Convención de nombres

- `📝_` = evaluación formal
- `(alumno)` = versión para resolver
- `📘_` = guía del profesor, criterio de corrección y enfoque didáctico

## Bloques actuales

### `📝_10_evaluacion_fundamentos`

Mide la base del curso: funciones, estructuras simples, condiciones y resumen global.

### `📝_13_evaluacion_datos_y_calidad`

Mide modelado, estado, excepciones, JSON, SQLite y organización del código.

### `📝_16_evaluacion_web_y_apis`

Mide rutas JSON, validación, códigos HTTP y consumo de respuestas.

## Cómo corregir con criterio

Lo importante es revisar:

- si el flujo general tiene sentido
- si las validaciones están en el orden correcto
- si el alumno reutiliza funciones o repite lógica
- si el estado cambia cuando debe cambiar
- si el código se entiende sin explicación constante

## Señales de que el bloque todavía no está maduro

Conviene volver a práctica o repaso si el alumno:

- mezcla demasiado en una sola función
- no distingue cálculo de presentación
- resuelve por copia, pero no sabe explicar el flujo
- se rompe ante cambios pequeños del enunciado
- no sabe qué validación debería ir primero

## Cómo usar la guía `📘_` de cada evaluación

La guía no está para leerla antes de resolver. Está pensada para:

- preparar la corrección
- dar feedback rápido
- detectar errores frecuentes
- orientar recuperación
- mantener un criterio uniforme entre bloques

## Cierre

Esta carpeta funciona bien cuando tiene pocas evaluaciones, muy claras, alineadas con el curso y fáciles de corregir con criterio.
